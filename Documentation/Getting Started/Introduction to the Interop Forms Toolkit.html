Interop Forms Toolkit
Introduction to the Interop Forms Toolkit

Overview

The Microsoft Interop Forms Toolkit is aimed at simplifying the use of Visual Basic.NET Windows Forms within a Visual Basic 6.0 forms application.  The toolkit is targeted specifically at providing tools and guidance for performing a phased migration of a Visual Basic 6.0 forms-based application to Visual Basic .NET.  The goal of a phased migration is a production release at the end of each phase that has both Visual Basic 6.0 and .NET forms running in the same Visual Basic 6.0 .exe process. 

What’s New in ver 2.1

  • Latest Visual Studio Support - the tools and Add-in have been upgraded to support several versions of Visual Studio: Visual Studio 2005, Visual Studio 2008, and Visual Studio 2010 along with fixes for a few reported bugs. 
  • No Runtime Redistributable Changes – to ensure compatibility with existing deployments the runtime redistributable has not been changed. However, developers may still make use of new language and .NET Framework features.

    What’s New in ver 2.0

  • Controls - The Interop Forms Toolkit now includes a template for Interop UserControls, which allow you to create controls in .NET that can be used on a Visual Basic 6.0 form. For more information, see Interop UserControls. 
  • MDI Support – Previously the Interop Forms Toolkit did not support MDI forms, however you can now extend your Visual Basic 6.0  MDI forms by using Interop UserControls.
  • Redistributable Component – The Interop Forms Toolkit Redistributable Package 2.0 makes it easier to create setup packages for your hybrid applications.
  • RegFree COM support – Interop UserControls automatically generate a RegFree COM manifest which allows for easier deployment of the .NET assemblies.

    Requirements

    This Toolkit requires Visual Basic 6.0 and a full version of Visual Studio 2005 or later (Note: Visual Studio AddIns are not supported in Visual Basic Express).
